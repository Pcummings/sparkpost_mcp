<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test process → SparkPost REST API | Tests must never cross this boundary; `globalThis.fetch` is replaced before any call so no request leaves the process. |
| module import → process env | `src/sparkpost.ts` reads `SPARKPOST_API_KEY` only inside `getHeaders()` at call time, never at import, so importing for tests neither requires nor exposes the key. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-IDISC | Information Disclosure | test/spRequest.test.ts | low | mitigate | Never set `SPARKPOST_API_KEY` in test env; monkeypatch `globalThis.fetch` so no real auth header is built or sent (getHeaders returns `""` when unset). |
| T-01-TAMP | Tampering | test fetch stub | low | mitigate | Every test replaces `globalThis.fetch` in `beforeEach` and restores in `afterEach`; no test path reaches the real SparkPost API. |

No threat reaches the `high` blocking threshold — the design is fully mocked and secret-free, so this plan is not blocked.
</threat_model>
